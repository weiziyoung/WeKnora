### 1. 数据库变更 (Database)
- 新建迁移文件 : 创建一个新的 SQL 迁移文件，在 messages 表中新增 feedback 字段。
  - 字段名: feedback
  - 类型: JSONB
  - 用途: 存储用户的反馈信息，结构灵活，便于后续扩展。
  - 数据结构示例: {"rating": "like", "comment": "回答不准确"}
### 2. 后端开发 (Backend)
- 数据模型 ( internal/types/message.go ) :
  - 修改 Message 结构体，增加 Feedback 字段。
  - 定义 Feedback 结构体，包含 Rating (枚举: like/dislike) 和 Comment (字符串)。
- Repository 层 ( internal/application/repository ) :
  - 在 MessageRepository 中增加 UpdateFeedback 方法，用于更新指定消息的反馈字段。
- Service 层 ( internal/application/service ) :
  - 在 MessageService 中增加业务逻辑，校验参数并调用 Repository 更新反馈。
- API 接口 ( internal/handler & internal/router ) :
  - 新增接口: PUT /api/v1/messages/:session_id/:message_id/feedback
  - Handler: 实现 UpdateFeedback 方法处理 HTTP 请求。
### 3. 前端开发 (Frontend)
- 组件修改 ( botmsg.vue ) :
    - 在 answer-toolbar (回答底部的工具栏) 中添加两个图标按钮：
      - 👍 点赞 (Like) : 点击后直接发送好评反馈，图标高亮显示。
      - 👎 点踩 (Dislike) : 点击后弹出模态框，标题“对结果不满意的原因”，包含以下选项：
        1. 回答遗漏明明存在的信息
        2. 推荐冗余，我不需要这些信息
        3. 回答存在事实错误， 或包含错误信息
        4. 答非所问
        5. 答案对我没有帮助
        6. 其他（带输入框，默认提示语：“欢迎反馈在使用中遇到的任何问题，感谢您的支持”）
      - 确认提交后，发送反馈数据（包括选中的原因和补充说明）。
  - API 客户端 ( src/api/chat ) :
  - 封装 updateMessageFeedback 方法对接后端接口。
### 4. 管理员查看反馈 (Admin View)

#### 4.1 用户权限管理
*   **数据库变更**:
    *   在 `users` 表中新增 `is_admin` 字段 (Boolean, default: false)。
    *   该字段用于标识系统管理员权限。
*   **后端 API**:
    *   更新登录接口 (`/auth/login`) 和用户信息接口 (`/auth/me`) 的响应，返回 `is_admin` 字段。

#### 4.2 管理员入口
*   **前端实现**:
    *   在全局 Store (`authStore`) 中保存用户的 `is_admin` 状态。
    *   修改侧边栏组件 (`menu.vue`)，根据 `is_admin` 状态动态渲染管理员专属菜单项（如“系统管理”或“反馈中心”）。
    *   该入口仅对 `is_admin=true` 的用户可见。

#### 4.3 反馈管理页面
*   **新页面**: 创建 `/admin/feedbacks` 页面。
*   **功能**:
    *   展示所有用户的反馈列表（分页）。
    *   支持按时间、反馈类型（点赞/点踩）、用户筛选。
    *   显示关联的会话 ID 和消息内容，方便回溯问题上下文。
*   **后端 API**:
    *   新增接口: `GET /api/v1/admin/feedbacks`
    *   权限控制: 仅允许管理员访问。

### 5. 交互流程
1.  用户在 AI 回答下方看到“点赞/点踩”按钮。 
2.  用户点击按钮，前端立即更新 UI 状态（高亮图标）。
3.  前端异步调用 API 将反馈数据发送给后端。
4.  后端更新数据库中的消息记录。
5.  管理员登录系统，侧边栏显示“反馈管理”入口。
6.  管理员进入页面，查看并处理用户反馈。

这个方案对现有架构影响最小，且利用 JSONB 字段保证了未来扩展性（例如增加更多反馈维度）。
